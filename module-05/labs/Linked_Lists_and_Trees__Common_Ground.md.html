Linked Lists and Trees: Common Ground

Pointers, References, and the Art of Connecting Data

Why This Matters

Arrays are powerful, but they have a hidden cost: everything has to live next to everything else. When your data needs to grow, shrink, or be reorganized frequently, that contiguity requirement becomes expensive. Linked structures — lists, trees, graphs — solve this by storing where the next piece of data lives alongside the data itself. Once you understand that idea, you can build almost any connected data structure.

This page covers:

  1. Array-based storage vs. linked storage
  2. The linked list as a foundation
  3. The binary tree as a natural extension

Part 1: Contiguous vs. Linked Storage

How an Array (or Python list / C++ vector) Works

When you create a standard list or vector, your program asks the operating system for a single, unbroken block of memory. Every element lives at a predictable offset from the start.

Python list: items = [10, 20, 30, 40, 50]

Memory (contiguous block):
┌────┬────┬────┬────┬────┐
│ 10 │ 20 │ 30 │ 40 │ 50 │
└────┴────┴────┴────┴────┘
  ^
  base address
  
items[0] = base + 0
items[2] = base + 2   ← instant calculation, O(1) access

This is why random access is O(1) — the computer does arithmetic, not searching.

The tradeoff: Insert or delete in the middle and every element after that point has to move.

Insert 99 at index 2:

Before:  [10, 20, 30, 40, 50]
Shift:   [10, 20, --, 30, 40, 50]   ← everything slides right
After:   [10, 20, 99, 30, 40, 50]

For large lists, this shifting is O(n). Resize past the allocated block, and the whole list may need to be copied to a new location.

How a Linked Structure Works

A linked structure gives up contiguous memory in exchange for flexible connectivity. Each element lives wherever the memory allocator puts it — potentially scattered all over RAM. Instead of relying on position, each element carries a pointer (C++) or reference (Python) to the next element.

Linked list: 10 → 20 → 30 → 40 → 50

Memory (scattered):
                        ┌──────────────┐
  ┌──────────┐          │              │          ┌──────────┐
  │ 10 │  ──────►  │ 20 │  ──────►  │ 30 │  ──────►  │ 40 │  ──────►  │ 50 │ None │
  └──────────┘          └──────────────┘          └──────────┘

  addr: 0x4a2           addr: 0x1f7               addr: 0x8c3

Each box is a node: it holds a value and a reference to wherever the next node happens to be living. The nodes do not need to be neighbors.

The tradeoff you’re accepting: You lose O(1) random access. To reach element 4, you must start at element 0 and follow the chain. Access is O(n).

What you gain: Insert or delete at any point requires only changing a couple of pointers — no shifting, no copying.

Insert 99 between 20 and 30:

Before:  10 → 20 → 30 → 40 → 50

Step 1: Create new node containing 99
Step 2: new_node.next = node_20.next   (point 99 → 30)
Step 3: node_20.next = new_node        (point 20 → 99)

After:   10 → 20 → 99 → 30 → 40 → 50

Only two pointer assignments. No elements moved.

Side-by-Side Comparison

Operation Array / Vector / Python list Linked List
Access by index O(1) O(n)
Insert/delete at front O(n) O(1)
Insert/delete at middle O(n) O(1) with pointer to node
Insert/delete at end O(1) amortized O(n) or O(1) with tail pointer
Memory layout Contiguous Scattered
Cache performance Excellent Poor
Resize cost Potential full copy None (nodes added individually)

Neither structure is better — they make different tradeoffs. Knowing which tradeoff you need is the skill.

Part 2: The Linked List

Structure

A singly linked list consists of nodes chained together. Each node contains:

  • A value (the data you care about)
  • A next pointer/reference (where to find the following node)

The list itself just needs to know where the head is. Everything else is reachable from there.

Singly Linked List

  head
   │
   ▼
┌───────┬──────┐    ┌───────┬──────┐    ┌───────┬──────┐
│  10   │  ●───────►│  20   │  ●───────►│  30   │ None │
└───────┴──────┘    └───────┴──────┘    └───────┴──────┘
  node_A              node_B              node_C (tail)

A doubly linked list adds a prev pointer so you can traverse in both directions:

Doubly Linked List

  head                                              tail
   │                                                 │
   ▼                                                 ▼
┌──────┬───┬───┐    ┌──────┬───┬───┐    ┌──────┬───┬───┐
│  10  │ ✗ │ ●─────►│  20  │ ◄─│ ●─────►│  30  │ ◄─│ ✗ │
└──────┴───┴───┘    └──────┴───┴───┘    └──────┴───┴───┘
  prev=None           prev  next           prev  next=None

Core Operations

Traversal — follow the chain until next is None/nullptr:

current = head
while current is not None:
    process current.value
    current = current.next

Insert at front — cheapest insertion, O(1):

new_node.next = head
head = new_node

Insert at position — walk to the target, rewire two pointers:

current = head
walk to node just before insertion point

new_node.next = current.next
current.next = new_node

Delete a node — bypass it:

# To delete node_B (current.next):
current.next = current.next.next
# node_B is now unreferenced; memory is reclaimed

Mental Model

Think of a linked list as a scavenger hunt: each clue tells you only where the next clue is. You cannot skip ahead — you have to follow the trail. The upside is that adding or removing a clue is trivial: you just update what the previous clue points to.

Part 3: The Binary Tree — Extending the Idea

From Two Pieces to Three

A linked list node has two pieces:

┌───────┬──────┐
│ value │ next │
└───────┴──────┘

A binary tree node has three pieces:

┌───────┬──────┬───────┐
│ value │ left │ right │
└───────┴──────┴───────┘

Instead of one path forward, each node can branch in two directions. That single change — adding a second pointer — transforms a linear chain into a hierarchical structure.

Structure

Binary Tree

             ┌───────────┐
             │    50     │   ← root
             └─────┬─────┘
           ┌───────┴───────┐
     ┌─────┴────┐     ┌────┴─────┐
     │    25    │     │    75    │
     └──┬───┬───┘     └───┬───┬──┘
      ┌─┴─┐ ┌─┴─┐       ┌─┴─┐ ┌─┴─┐
      │10 │ │30 │       │60 │ │90 │
      └───┘ └───┘       └───┘ └───┘

Leaves (nodes with no children) have left=None, right=None

Vocabulary:

Term Meaning
Root The top node; has no parent
Child A node directly below another
Parent A node directly above another
Leaf A node with no children
Height Longest path from root to any leaf
Subtree A node plus all its descendants

Why Branching Matters for Performance

In a linked list, finding a value takes O(n) in the worst case — you might have to check every node.

In a well-organized binary tree (specifically a Binary Search Tree), each step eliminates half the remaining candidates:

Search for 60 in the BST above:

Start at root (50). Is 60 > 50? Yes → go right.
At 75. Is 60 < 75? Yes → go left.
At 60. Found it!

3 comparisons for a 7-node tree. 
As the tree grows, search time grows as O(log n) not O(n).

This is the same intuition as binary search on a sorted array — but now the structure enforces the organization rather than requiring a pre-sorted array.

The Binary Search Tree Property

A BST maintains one invariant at every node:

All values in the LEFT subtree < node value < All values in the RIGHT subtree

      ┌───┐
      │ 8 │
      └─┬─┘
    ┌───┴───┐
  ┌─┴─┐   ┌─┴─┐
  │ 3 │   │10 │
  └─┬─┘   └─┬─┘
  ┌─┴─┐     └──┐
  │ 1 │       ┌┴──┐
  └───┘       │14 │
              └───┘

3 < 8 ✓   10 > 8 ✓   1 < 3 ✓   14 > 10 ✓

Mental Model

If a linked list is a scavenger hunt (one path forward), a binary tree is a decision tree: at every node you make a yes/no choice — left or right — and the structure shrinks your remaining search space with each step.

The pointer mechanics are identical to a linked list. The difference is purely in how many pointers each node carries and what organizational rules you enforce.

Connecting the Concepts

Array:         Fixed memory, index arithmetic, fast access, slow insert/delete

Linked List:   Scattered memory, pointer chains
               [value | next] → [value | next] → [value | None]

Binary Tree:   Scattered memory, pointer branches
               [value | left | right]
                        ↙          ↘
               [value | left | right]  [value | left | right]

Every data structure you study this semester — stacks, queues, heaps, BSTs, AVL trees — is built on one of these foundations. The pointer/reference mechanism is the same. What changes is how many pointers, what rules govern them, and which operations you optimize for.

Python Implementation

# ============================================================
# Linked List and Binary Tree — Python Reference Implementation
# CSC 249 — Data Structures
# ============================================================


# ─────────────────────────────────────────────
# PART 1: SINGLY LINKED LIST
# ─────────────────────────────────────────────

class ListNode:
    """A single node in a singly linked list."""
    def __init__(self, value):
        self.value = value
        self.next = None   # Reference to the next node (or None)


class LinkedList:
    """Singly linked list with basic operations."""

    def __init__(self):
        self.head = None   # The entry point to the list

    # ── Traversal ──────────────────────────────
    def display(self):
        """Print the list from head to tail."""
        elements = []
        current = self.head
        while current is not None:
            elements.append(str(current.value))
            current = current.next
        print(" → ".join(elements) if elements else "(empty list)")

    # ── Insertion ──────────────────────────────
    def insert_front(self, value):
        """Insert a new node at the front — O(1)."""
        new_node = ListNode(value)
        new_node.next = self.head
        self.head = new_node

    def insert_end(self, value):
        """Insert a new node at the end — O(n)."""
        new_node = ListNode(value)
        if self.head is None:
            self.head = new_node
            return
        current = self.head
        while current.next is not None:   # Walk to the last node
            current = current.next
        current.next = new_node

    # ── Search ─────────────────────────────────
    def search(self, target):
        """Return True if target exists in the list — O(n)."""
        current = self.head
        while current is not None:
            if current.value == target:
                return True
            current = current.next
        return False

    # ── Deletion ───────────────────────────────
    def delete(self, target):
        """Remove the first node containing target — O(n)."""
        if self.head is None:
            return

        # Special case: deleting the head
        if self.head.value == target:
            self.head = self.head.next
            return

        current = self.head
        while current.next is not None:
            if current.next.value == target:
                current.next = current.next.next   # Bypass the target node
                return
            current = current.next


# ─────────────────────────────────────────────
# PART 2: BINARY SEARCH TREE
# ─────────────────────────────────────────────

class TreeNode:
    """A single node in a binary tree.
    
    Unlike a list node (one pointer), a tree node has TWO child pointers.
    """
    def __init__(self, value):
        self.value = value
        self.left = None    # Reference to the left child subtree
        self.right = None   # Reference to the right child subtree


class BinarySearchTree:
    """Binary Search Tree with insert, search, and traversal."""

    def __init__(self):
        self.root = None

    # ── Insertion ──────────────────────────────
    def insert(self, value):
        """Public insert — delegates to recursive helper."""
        self.root = self._insert_recursive(self.root, value)

    def _insert_recursive(self, node, value):
        """Recursively find the correct position and insert."""
        if node is None:
            return TreeNode(value)       # Base case: empty spot found

        if value < node.value:
            node.left = self._insert_recursive(node.left, value)
        elif value > node.value:
            node.right = self._insert_recursive(node.right, value)
        # If value == node.value, we skip (no duplicates in this implementation)

        return node

    # ── Search ─────────────────────────────────
    def search(self, target):
        """Return True if target exists in the tree — O(log n) average."""
        return self._search_recursive(self.root, target)

    def _search_recursive(self, node, target):
        if node is None:
            return False                 # Fell off the tree: not found
        if target == node.value:
            return True
        elif target < node.value:
            return self._search_recursive(node.left, target)
        else:
            return self._search_recursive(node.right, target)

    # ── Traversal ──────────────────────────────
    def inorder(self):
        """In-order traversal: left → node → right.
        
        For a BST, this visits nodes in ascending sorted order.
        """
        result = []
        self._inorder_recursive(self.root, result)
        return result

    def _inorder_recursive(self, node, result):
        if node is None:
            return
        self._inorder_recursive(node.left, result)
        result.append(node.value)
        self._inorder_recursive(node.right, result)


# ─────────────────────────────────────────────
# DEMO
# ─────────────────────────────────────────────

if __name__ == "__main__":

    print("=== LINKED LIST DEMO ===")
    ll = LinkedList()
    for val in [10, 20, 30, 40, 50]:
        ll.insert_end(val)
    ll.display()                         # 10 → 20 → 30 → 40 → 50

    ll.insert_front(5)
    ll.display()                         # 5 → 10 → 20 → 30 → 40 → 50

    ll.delete(30)
    ll.display()                         # 5 → 10 → 20 → 40 → 50

    print("Search 20:", ll.search(20))   # True
    print("Search 99:", ll.search(99))   # False

    print("\n=== BINARY SEARCH TREE DEMO ===")
    bst = BinarySearchTree()
    for val in [50, 25, 75, 10, 30, 60, 90]:
        bst.insert(val)

    print("In-order traversal:", bst.inorder())   # [10, 25, 30, 50, 60, 75, 90]
    print("Search 60:", bst.search(60))           # True
    print("Search 45:", bst.search(45))           # False

C++ Implementation

// ============================================================
// Linked List and Binary Tree — C++ Reference Implementation
// CSC 249 — Data Structures
// ============================================================
//
// Key C++ concepts illustrated here:
//   - Raw pointers (node* next, node* left, node* right)
//   - Dynamic allocation (new) and deallocation (delete)
//   - nullptr for empty pointers (equivalent to Python's None)
//   - Struct for simple data holders (no private members needed)
// ============================================================

#include <iostream>
#include <vector>
using namespace std;


// ─────────────────────────────────────────────
// PART 1: SINGLY LINKED LIST
// ─────────────────────────────────────────────

struct ListNode {
    int value;
    ListNode* next;   // Pointer to the next node in memory (anywhere in RAM)

    ListNode(int val) : value(val), next(nullptr) {}
};


class LinkedList {
private:
    ListNode* head;

public:
    LinkedList() : head(nullptr) {}

    // ── Destructor ─────────────────────────────
    // C++ does NOT have garbage collection.
    // We must manually free every node we allocated with 'new'.
    ~LinkedList() {
        ListNode* current = head;
        while (current != nullptr) {
            ListNode* next = current->next;
            delete current;        // Return this node's memory to the OS
            current = next;
        }
    }

    // ── Traversal ──────────────────────────────
    void display() const {
        ListNode* current = head;
        while (current != nullptr) {
            cout << current->value;
            if (current->next != nullptr) cout << " → ";
            current = current->next;
        }
        cout << endl;
    }

    // ── Insertion ──────────────────────────────
    void insertFront(int value) {
        ListNode* newNode = new ListNode(value);  // Allocate on the heap
        newNode->next = head;
        head = newNode;
    }

    void insertEnd(int value) {
        ListNode* newNode = new ListNode(value);
        if (head == nullptr) {
            head = newNode;
            return;
        }
        ListNode* current = head;
        while (current->next != nullptr) {
            current = current->next;
        }
        current->next = newNode;
    }

    // ── Search ─────────────────────────────────
    bool search(int target) const {
        ListNode* current = head;
        while (current != nullptr) {
            if (current->value == target) return true;
            current = current->next;
        }
        return false;
    }

    // ── Deletion ───────────────────────────────
    void deleteValue(int target) {
        if (head == nullptr) return;

        if (head->value == target) {
            ListNode* toDelete = head;
            head = head->next;
            delete toDelete;       // Must explicitly free the memory
            return;
        }

        ListNode* current = head;
        while (current->next != nullptr) {
            if (current->next->value == target) {
                ListNode* toDelete = current->next;
                current->next = current->next->next;  // Bypass
                delete toDelete;
                return;
            }
            current = current->next;
        }
    }
};


// ─────────────────────────────────────────────
// PART 2: BINARY SEARCH TREE
// ─────────────────────────────────────────────

struct TreeNode {
    int value;
    TreeNode* left;    // Pointer to left child subtree
    TreeNode* right;   // Pointer to right child subtree

    TreeNode(int val) : value(val), left(nullptr), right(nullptr) {}
};


class BinarySearchTree {
private:
    TreeNode* root;

    // ── Private recursive helpers ───────────────
    TreeNode* insertRecursive(TreeNode* node, int value) {
        if (node == nullptr) {
            return new TreeNode(value);   // Base case: empty slot
        }
        if (value < node->value) {
            node->left = insertRecursive(node->left, value);
        } else if (value > node->value) {
            node->right = insertRecursive(node->right, value);
        }
        return node;
    }

    bool searchRecursive(TreeNode* node, int target) const {
        if (node == nullptr) return false;
        if (target == node->value) return true;
        if (target < node->value)
            return searchRecursive(node->left, target);
        return searchRecursive(node->right, target);
    }

    void inorderRecursive(TreeNode* node, vector<int>& result) const {
        if (node == nullptr) return;
        inorderRecursive(node->left, result);
        result.push_back(node->value);
        inorderRecursive(node->right, result);
    }

    // Free all nodes (post-order: children before parent)
    void destroyTree(TreeNode* node) {
        if (node == nullptr) return;
        destroyTree(node->left);
        destroyTree(node->right);
        delete node;
    }

public:
    BinarySearchTree() : root(nullptr) {}

    ~BinarySearchTree() {
        destroyTree(root);
    }

    // ── Public interface ───────────────────────
    void insert(int value) {
        root = insertRecursive(root, value);
    }

    bool search(int target) const {
        return searchRecursive(root, target);
    }

    vector<int> inorder() const {
        vector<int> result;
        inorderRecursive(root, result);
        return result;
    }
};


// ─────────────────────────────────────────────
// DEMO
// ─────────────────────────────────────────────

int main() {

    cout << "=== LINKED LIST DEMO ===" << endl;
    LinkedList ll;
    for (int val : {10, 20, 30, 40, 50}) {
        ll.insertEnd(val);
    }
    ll.display();              // 10 → 20 → 30 → 40 → 50

    ll.insertFront(5);
    ll.display();              // 5 → 10 → 20 → 30 → 40 → 50

    ll.deleteValue(30);
    ll.display();              // 5 → 10 → 20 → 40 → 50

    cout << "Search 20: " << (ll.search(20) ? "true" : "false") << endl;
    cout << "Search 99: " << (ll.search(99) ? "true" : "false") << endl;

    cout << "\n=== BINARY SEARCH TREE DEMO ===" << endl;
    BinarySearchTree bst;
    for (int val : {50, 25, 75, 10, 30, 60, 90}) {
        bst.insert(val);
    }

    cout << "In-order traversal: ";
    for (int val : bst.inorder()) {
        cout << val << " ";               // 10 25 30 50 60 75 90
    }
    cout << endl;

    cout << "Search 60: " << (bst.search(60) ? "true" : "false") << endl;
    cout << "Search 45: " << (bst.search(45) ? "true" : "false") << endl;

    return 0;
    // Destructors for ll and bst run here automatically,
    // freeing all heap-allocated nodes.
}
