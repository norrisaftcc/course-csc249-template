Graphs: Nodes, Edges, Worlds

The data structure that models connections — maps, networks, game worlds, relationships, and everything in between.

What Is a Graph?

A graph is two things:

Vertices (Nodes)

( A ) ( B ) ( C ) ( D )

Things. Places. People. Concepts.

Edges (Connections)

( A )---( B ) | | ( C )---( D )

Links. Roads. Friendships. Wires.

That's it. Everything else is detail.

Graphs Are Everywhere

Maps & Navigation

Cities = vertices. Roads = edges.
Google Maps is a graph algorithm.

Social Networks

People = vertices. Friendships = edges.
Facebook has 3 billion nodes.

Game Worlds

Rooms = vertices. Doors = edges.
Zork, Skyrim, and your homework.

The Internet

Pages = vertices. Links = edges.
PageRank made Google a verb.

Directed vs. Undirected

Undirected

Tokyo ──── London

Connection goes both ways.
If I can fly Tokyo→London,
I can fly London→Tokyo.

Roads, friendships, flight routes

Directed

Junior ────▶ Senior

Connection is one-way.
You can be promoted,
but you can't un-promote.

One-way streets, Twitter follows, prerequisites

Weighted Edges

Edges can carry information — a label, a cost, a distance, a description.

( Tokyo )──── 6743 mi ────( New York ) │ │ 5941 mi 3425 mi │ │ ( London )──── 10562 mi ───( Sydney )

In our code, edge weights are strings. This keeps things flexible — "6743 miles" for distances, "north" for directions, "married to" for relationships. One graph class handles all cases.

Paths and Cycles

Path: A → B → D

( A )───▶( B ) │ ▼ ( C ) ( D )

A sequence of vertices
connected by edges.
"How do I get from here to there?"

Cycle: A → B → D → A

( A )───▶( B ) ▲ │ │ ▼ └─────( D )

A path that ends
where it started.
"Can I get back home?"

How Code Stores a Graph

The Adjacency List

( A )───( B ) │ │ ( C )───( D ) 
A → [ B, C ]
B → [ A, D ]
C → [ A, D ]
D → [ B, C ]

Each vertex keeps a list of its neighbors.
That's all an adjacency list is.

In our Graph class, this is an unordered_map (C++) or dict (Python) — the vertex is the key, its edge list is the value.

Building a Graph in Code

C++

Graph flights;

Vertex* tok = flights.AddVertex("Tokyo");
Vertex* nyc = flights.AddVertex("New York");
Vertex* lon = flights.AddVertex("London");

flights.AddUndirectedEdge(
    tok, nyc, "6743 miles");
flights.AddUndirectedEdge(
    tok, lon, "5941 miles");
flights.AddUndirectedEdge(
    nyc, lon, "3425 miles");

Python

flights = Graph()

tok = flights.add_vertex("Tokyo")
nyc = flights.add_vertex("New York")
lon = flights.add_vertex("London")

flights.add_undirected_edge(
    tok, nyc, "6743 miles")
flights.add_undirected_edge(
    tok, lon, "5941 miles")
flights.add_undirected_edge(
    nyc, lon, "3425 miles")

Same structure, same operations, two languages.

Asking Questions About a Graph

// "What flights leave from Tokyo?"
for (Edge* e : *graph.GetEdgesFrom(tokyo)) {
    cout << e->toVertex->label      // destination
         << ", " << e->weight;       // distance
}

// "Can I fly direct from London to Sydney?"
if (graph.HasEdge(london, sydney)) { ... }

// "How many cities are in the network?"
graph.GetVertices().size();

The graph API gives you the building blocks. You ask the questions.

Graphs as Game Worlds

north [ Kitchen ]─────────[ Living Room ] │ │ south east │ │ [ Garage ] [ Bedroom ] │ down │ [ Basement ]

Every classic text adventure — Zork, Colossal Cave, MUDs — is a graph. Rooms are vertices. Doors are edges. Edge labels are directions. When you type go north, the engine follows an edge.
