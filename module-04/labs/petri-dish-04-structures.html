Stack (LIFO)

size: 0
Push: O(1) | Pop: O(1) | Last in = First out

Queue (FIFO)

size: 0
Enqueue: O(1) | Dequeue: O(1) | First in = First out

BST (Sorted Tree)

size: 0
Insert: O(log N) avg | Search: O(log N) avg

What You're Seeing

Stack (LIFO): Items stack vertically. Push adds to the top. Pop removes from the top. The most recent item is always the one that leaves first — like a stack of plates.

Queue (FIFO): Items line up horizontally. Enqueue adds to the right. Dequeue removes from the left. The oldest item is always processed first — like a line at a store.

BST: Items are organized in a tree by value. Left children have smaller values, right children have larger. Search follows a path from root, going left or right at each node. Balanced trees give O(log N) search. Try "Insert 1-10 (sorted)" to see what happens when data arrives in order — the tree degenerates into a linked list!
